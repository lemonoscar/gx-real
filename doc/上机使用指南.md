# 上机使用指南

这份流程对应当前 `gx-real` 真机部署链路：

- Go2 腿部由 `12D` RL policy 控制
- X5 机械臂保持 `--arm_pose` 指定的 6 关节姿态
- Go2 底层走 `lowstate/lowcmd`
- X5 底层走 `can0`
- 默认使用 `internal` 起身流程，不走手动接管流程
- 腿部低层位置控制 Kp 固定为 `100`

当前版本的关键行为：

- 按 `R1`：启动 internal 起身，即使你已经先让狗站起来，也按这个流程执行
- 等 internal 起身完成后按 `L2`：先低层对齐，再进入 RL rollout
- 速度命令先从 `0 0 0` 接管，再自动 ramp 到 `--cmd-vx/--cmd-vy/--cmd-yaw`
- `--arm_pose` 会贯穿 internal 起身、对齐、pose test 和 rollout，不再被默认机械臂姿态覆盖
- 默认要求收到可用的 `sport_mode` 状态才允许低层接管

## 1. 第一次上机前

### 1.1 退出 conda

```bash
conda deactivate
```

### 1.2 确认 policy 存在

```bash
ls ~/gx-real/policies/policy.onnx
```

### 1.3 安装 ARX5 Python 接口

只需要第一次做：

```bash
cd ~/gx-real/arx5-sdk
source /opt/ros/foxy/setup.bash
/usr/bin/python3 -m pip install --user --no-build-isolation .
```

检查：

```bash
cd ~/gx-real
source scripts/setup_env.sh
/usr/bin/python3 -c "import arx5_interface; print('arx5 ok')"
```

### 1.4 编译 Unitree ROS2 消息包

只需要第一次做：

```bash
cd ~/gx-real/unitree_ros2/cyclonedds_ws
source /opt/ros/foxy/setup.bash
colcon build
```

只要 `unitree_go / unitree_api / unitree_hg` 编出来即可。

## 2. 每次上机前

### 2.1 进入仓库并加载环境

```bash
conda deactivate
cd ~/gx-real
git pull
source scripts/setup_env.sh
```

### 2.2 检查 Python/ROS 环境

```bash
scripts/check_env.sh
```

通过时应看到：

```text
[gx-real] python imports OK
```

### 2.3 检查并激活 `can0`

```bash
ip -details link show can0
```

如果已经是 `state UP`，继续。
如果没有 `can0` 或接口没起来：

```bash
scripts/setup_arx_can.sh
ip -details link show can0
```

### 2.4 关闭 Go2 `sport_mode`

Go2 原厂高层控制和低层 `lowcmd` 不能同时抢控制权。上机前先关闭：

```bash
~/unitree_sdk2/build/disable_sports_mode_go2 eth0
```

如果网卡不是 `eth0`，换成实际连接 Go2 的网卡名。程序输出 `sport_mode is deactivate` 后再继续。

## 3. 标准启动命令

你现在要的机械臂姿态是：

```text
0 0.5 0.3 0 0 0
```

你现在要的速度命令是：

```text
0.5 0 0
```

对应启动命令：

```bash
cd ~/gx-real
scripts/run_leg12_real.sh \
  --device cpu \
  --pose_estimator none \
  --standup-mode internal \
  --cmd-vx 0.5 \
  --cmd-vy 0.0 \
  --cmd-yaw 0.0 \
  --gripper-cmd 0.0 \
  --arm_pose 0.0 0.5 0.3 0.0 0.0 0.0
```

看到：

```text
Deploy node ready
```

然后按：

1. `R1`：启动 internal 起身
2. 等待 internal 起身结束，狗稳定站住
3. `L2`：启动低层对齐，然后进入 RL rollout

预期日志会包含：

```text
Runtime targets | standup_mode=internal arm_pose_source=user arm_hold_pose=[0.000 0.500 0.300 0.000 0.000 0.000] commanded_leg_kp=[100.0 ...
Starting dog-only startup before rollout
Dog-only startup completed; starting rollout
Policy diag | ... commands=[0.500 0.000 0.000] ... lowcmd_kp=[100.000 ...
```

## 4. 更保守的验证命令

第一次真机建议先把 `--cmd-vx` 改成 `0.0` 或 `0.1`，确认低层接管和机械臂覆盖正常后再上 `0.5`。

静态接管：

```bash
scripts/run_leg12_real.sh \
  --device cpu \
  --pose_estimator none \
  --standup-mode internal \
  --cmd-vx 0.0 \
  --cmd-vy 0.0 \
  --cmd-yaw 0.0 \
  --gripper-cmd 0.0 \
  --arm_pose 0.0 0.5 0.3 0.0 0.0 0.0
```

低速前进：

```bash
scripts/run_leg12_real.sh \
  --device cpu \
  --pose_estimator none \
  --standup-mode internal \
  --cmd-vx 0.1 \
  --cmd-vy 0.0 \
  --cmd-yaw 0.0 \
  --gripper-cmd 0.0 \
  --arm_pose 0.0 0.5 0.3 0.0 0.0 0.0
```

操作仍然是：看到 `Deploy node ready` 后按 `R1`，等待起身完成，再按 `L2`。

## 5. 手柄按键

- `R1`：启动 internal 起身
- `L2`：起身结束后，启动低层对齐并进入 RL rollout
- `R2`：停止 RL policy
- `L1`：紧急停止并退出程序

## 6. `pose_test` 诊断流程

如果怀疑低层接管有问题，可以先只做关节目标跟踪测试：

```bash
scripts/run_leg12_real.sh \
  --device cpu \
  --pose_estimator none \
  --standup-mode pose_test \
  --cmd-vx 0.0 \
  --cmd-vy 0.0 \
  --cmd-yaw 0.0 \
  --gripper-cmd 0.0 \
  --arm_pose 0.0 0.5 0.3 0.0 0.0 0.0
```

看到 `Deploy node ready` 后按一次 `L2`。

重点看日志：

```text
Pose test diag | ... max_leg_error=... current_leg_q=... lowcmd_leg_q_policy=...
```

判断：

- `max_leg_error` 能降到很小：低层位置控制基本可用
- `max_leg_error` 长期大于 `0.08 rad`：不要跑动态 policy，先排查 sport mode、低层通信、站姿和关节顺序

## 7. 常见问题

### 7.1 程序启动后机器人不动

看到 `Deploy node ready` 只表示节点启动成功。还需要：

1. `sport_mode` 已关闭或确认 idle
2. 按一次 `R1`
3. 等 internal 起身完成
4. 再按一次 `L2`

### 7.2 按 `L2` 后拒绝启动

如果看到：

```text
Refusing low-level rollout because sport_mode state has not been received
```

说明程序没有收到可用的 `sport_mode` 状态。先检查 ROS2 状态链路。只有受控诊断时才加：

```bash
--allow-unknown-sport-mode
```

如果看到：

```text
Refusing low-level rollout while sport_mode is still active
```

先执行：

```bash
~/unitree_sdk2/build/disable_sports_mode_go2 eth0
```

### 7.3 `commands` 非零但狗不前进

先看 `Policy diag` 里的信息：

- `commands`：是否已经是非零，例如 `[0.500 0.000 0.000]`
- `lowcmd_leg_q_policy / lowcmd_leg_q_hw`：程序是否确实在发腿部目标
- `lowcmd_kp`：是否是 `[100.000 ...]`
- `current_leg_q / leg_q_error`：真实关节是否跟上目标

如果 `lowcmd_kp=100`、`lowcmd` 在变，但 `current_leg_q` 仍明显跟不上，优先排查低层控制权、sport mode 冲突、力矩限制、电池电量和关节顺序。

### 7.4 机械臂没有到 `--arm_pose`

确认启动日志里有：

```text
arm_hold_pose=[0.000 0.500 0.300 0.000 0.000 0.000]
```

如果日志正确但机械臂不动，优先排查：

- `can0` 是否 `UP`
- `arx5_interface` 是否正常 import
- 是否误加了 `--disable-arm`
- X5 是否供电并连接到同一个 CAN

## 8. 推荐上机顺序摘要

1. `conda deactivate`
2. `cd ~/gx-real && git pull && source scripts/setup_env.sh`
3. `scripts/check_env.sh`
4. `scripts/setup_arx_can.sh`
5. `~/unitree_sdk2/build/disable_sports_mode_go2 eth0`
6. 启动 `scripts/run_leg12_real.sh ... --standup-mode internal ...`
7. 看到 `Deploy node ready`
8. 按 `R1`
9. 等狗稳定站住
10. 按 `L2`
11. 看 `Policy diag` 里的 `commands`、`lowcmd_kp`、`current_leg_q`
